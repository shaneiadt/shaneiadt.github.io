Further Configuration

We can configure our dev server a number of different ways. Here’s a couple examples:

  • openPage: which page to navigate to first when your browser pops open.
  • publicPath: where our static files will be served & can be accessed from.
  • https: we can enable https support.
  • headers: add additional headers to all responses.

These are only a few options, for more read the docs, at the moment let’s just set our output folder & port number.
